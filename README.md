# 影像處理工具 / Image Processing Tool

Python 影像處理專案，提供多種常見的影像處理功能。  
支援**線上瀏覽器版**（免安裝即用）、**本機桌面版**（PyQt6），以及**一鍵安裝工具**。  
所有程式碼皆附有逐行繁體中文說明，適合學習與二次開發。

---

## 功能列表

### 色彩空間分離

| 方法 | 說明 | 可調整參數 |
|------|------|-----------|
| 色彩空間分離 | 將影像轉換至指定色彩空間後，提取單一通道作為灰階影像輸出 | 色彩空間（BGR / HSV / Lab / YCrCb / HLS / LUV / XYZ / YUV / GRAY / RGB）、通道（0 / 1 / 2） |

---

### 二值化

將灰階影像轉成黑白二值影像，每種方法適用不同情境。

| 方法 | 說明 | 可調整參數 |
|------|------|-----------|
| 自訂門檻 | 手動設定亮度門檻值，高於門檻為白、低於為黑 | 門檻值（0～255）、黑白反轉 |
| Otsu 自動 | 自動計算最佳門檻，適合前景與背景分明的影像 | 黑白反轉 |
| 自適應 | 依每個像素的局部鄰域亮度各自決定門檻，適合照明不均的影像 | BlockSize（鄰域大小）、C 值（門檻偏移）、黑白反轉 |
| 區域二值化 | 將影像切成 列×欄 的格子，每格各自用 Otsu 計算最佳門檻後拼回，適合不同區域亮度差異大的影像 | 列數（1～8）、欄數（1～8）、黑白反轉 |

> **建議**：選擇二值化方法時，先用「Otsu 自動」嘗試，若效果不佳再切換。照明不均→自適應；區域亮度差異大→區域二值化。

---

### 物件萃取

| 方法 | 說明 | 可調整參數 |
|------|------|-----------|
| 白色物件框選 | 在二值影像中找出灰階值為 255 的白色相連區塊，計算數量並用外接形狀標示 | 外接形狀（最小外接矩形 / 正外接矩形 / 外接圓 / 凸包）、框線顏色、線寬、最小面積（過濾雜訊） |

> 輸出影像左上角會標示偵測到的物件總數（`Objects: N`）。  
> **建議流程**：先「二值化」→ 再「物件萃取」。

---

### 形態學運算

對二值影像進行結構元素的膨脹或侵蝕，可修補孔洞、去除雜點。

| 方法 | 說明 | 可調整參數 |
|------|------|-----------|
| 膨脹 | 擴張白色區域，可填補小孔洞、加粗線條 | 核心大小（0～20）、次數（1～10） |
| 侵蝕 | 縮小白色區域，可去除細小白點、細化線條 | 核心大小（0～20）、次數（1～10） |
| 開運算 | 先侵蝕再膨脹，去除小白點（雜訊）同時保留大物件 | 核心大小（0～20） |
| 閉運算 | 先膨脹再侵蝕，填補白色物件內部的小孔洞 | 核心大小（0～20） |

> **建議流程**：先「二值化」→ 再接形態學運算。

---

### 影像銳化

強調影像邊緣與細節，讓影像視覺上更清晰。

| 方法 | 說明 | 可調整參數 |
|------|------|-----------|
| Unsharp Mask | 先模糊再疊加差值，模擬傳統暗房銳化效果 | 強度（1～30）、半徑（1～15） |
| Laplacian 銳化 | 用 Laplacian 高通核心偵測邊緣並加強 | 強度（1～5） |

---

### 邊緣偵測

找出影像中亮度變化明顯的邊界。

| 方法 | 說明 | 可調整參數 |
|------|------|-----------|
| Canny | 多階段邊緣偵測，結果乾淨，最常用 | 低門檻（0～255）、高門檻（0～255） |
| Sobel | 分別計算 X/Y 方向梯度再合併，對噪音較敏感 | Kernel Size（步進索引） |
| Laplacian | 二階微分偵測，對影像噪音敏感 | 無 |
| Scharr | Sobel 的改良版，Kernel Size 固定為 3，精度較高 | 無 |

---

### 強度轉換

調整影像整體亮度與對比度。

| 方法 | 說明 | 可調整參數 |
|------|------|-----------|
| Gamma 校正 | 用冪次函數調整亮度，γ < 1 提亮、γ > 1 壓暗 | Gamma 值（0.1～5.0） |
| 對數轉換 | 壓縮高亮度範圍，讓暗部細節更明顯 | 縮放比例（c） |
| 線性轉換 | 線性調整亮度（α × I + β） | α 增益（0.1～5.0）、β 偏移（-100～100） |
| 直方圖等化 | 將像素分佈均勻化，自動提升對比 | 無 |
| CLAHE | 限制對比的自適應直方圖等化，避免過度增強 | ClipLimit（1.0～10.0）、TileGridSize（1～16） |

---

### 影像校正

| 方法 | 說明 | 可調整參數 |
|------|------|-----------|
| 仿射變換（偏移校正） | 對影像進行水平/垂直平移，用於補正拍攝偏移 | X 偏移（-100～100 px）、Y 偏移（-100～100 px） |

---

### 幾何轉換

改變影像的大小、方向或位置。

| 方法 | 說明 | 可調整參數 |
|------|------|-----------|
| 縮放 | 獨立調整 X/Y 方向縮放比例 | X 縮放%（10～300）、Y 縮放%（10～300） |
| 旋轉 | 以影像中心旋轉，自動擴展畫布保留完整影像（黑色填邊） | 角度（-180～180 度） |
| 平移 | 水平/垂直方向平移，超出邊界部分填黑 | X 平移（-300～300 px）、Y 平移（-300～300 px） |
| 翻轉 | 水平翻轉、垂直翻轉、或同時翻轉兩軸 | 翻轉方向（水平 / 垂直 / 兩軸） |

---

### 各種濾波

對影像進行平滑或去噪處理。

| 方法 | 說明 | 可調整參數 |
|------|------|-----------|
| 高斯模糊 | 以高斯加權核心模糊，去除高頻雜訊 | 核心大小（0～15）、Sigma（0.1～5.0） |
| 均值模糊 | 取鄰域平均值，最簡單的平滑濾波 | 核心大小（0～15） |
| 中值濾波 | 取鄰域中位數，對椒鹽雜訊效果最佳 | 核心大小（0～15） |
| 雙邊濾波 | 保留邊緣的平滑濾波，速度較慢但效果最好 | 直徑（1～30）、顏色 Sigma（1～150）、空間 Sigma（1～150） |

> 濾波函數支援彩色或灰階輸入，不強制轉灰階。

## 鏈式處理（Pipeline）

支援將多個方法**依序串接**，讓每一步的輸出成為下一步的輸入。

### 操作流程

```
選擇方法 → 調整參數 → 即時預覽效果 → 按「套用」→ 繼續疊加下一步
```

### 介面說明

| 區域 | 說明 |
|------|------|
| 當前影像 | 已套用所有步驟後的結果，每次套用後自動更新 |
| 效果預覽 | 即時顯示目前參數的效果，套用前可先確認 |
| 步驟歷史 | 顯示已套用的每個步驟名稱與參數值 |

### 輔助功能

| 功能 | 說明 |
|------|------|
| 復原上一步 | 取消最後一個套用的步驟，影像回到上一步的狀態 |
| 重設為原圖 | 清除所有步驟，從原始上傳影像重新開始 |
| 儲存結果 | 輸出最終影像（線上版：下載按鈕；桌面版：自動儲存到 `Pipeline_Result/`） |

## 如何新增影像處理方法

本專案採用「方法註冊表（Registry）」架構，**新增方法只需兩步，兩個 UI 自動更新**：

### 步驟 1：在 `Image_Processing/processing/` 新增處理函式

建立或在現有 `_fn.py` 中加入純函式：

```python
# 範例：Image_Processing/processing/my_fn.py
import cv2
import numpy as np

def my_method(image: np.ndarray, param_a: int, param_b: float) -> np.ndarray:
    # 影像處理邏輯
    result = ...
    return result
```

### 步驟 2：在 `registry.py` 的 `METHODS` 清單加入一筆

```python
# Image_Processing/processing/registry.py
from processing.my_fn import my_method

Method(
    id="my_method",
    category="我的分類",       # 現有分類或新分類皆可；同分類的方法會聚在一起
    name="我的方法名稱",
    description="方法說明（顯示在 UI 提示中）",
    params=[
        ParamSpec("param_a", "參數A名稱", "int",
                  default=50, min=0, max=100, step=1,
                  help="此參數的用途說明"),
        ParamSpec("param_b", "參數B名稱", "float",
                  default=1.0, min=0.1, max=5.0, step=0.1,
                  help="此參數的用途說明"),
    ],
    fn=lambda img, p: my_method(img, p["param_a"], p["param_b"]),
    output_folder="My_Output",
),
```

完成後，兩個 UI（桌面版與網頁版）都會**自動出現**此方法，可調整參數並加入鏈式處理流程，不需修改任何 UI 程式碼。

### ParamSpec 參數型別說明

| kind | 說明 | 必要欄位 |
|------|------|---------|
| `"int"` | 整數滑桿 | min, max, step |
| `"float"` | 浮點數滑桿 | min, max, step |
| `"choice"` | 下拉選單 | choices（list of (value, 顯示名)） |
| `"buttons"` | 按鈕群（radio） | choices（list of (value, 顯示名)） |

## 技術架構說明

### Registry 模式

`registry.py` 定義了兩個核心資料類別：

- **`ParamSpec`**：描述單一參數的規格（類型、範圍、預設值、說明）
- **`Method`**：描述一個完整的處理方法（分類、名稱、參數清單、執行函式、輸出資料夾）

兩個 UI 都讀取同一份 `METHODS` 清單，依分類分組呈現方法，依 `ParamSpec` 自動生成對應控制元件（滑桿、下拉選單、按鈕群）。

### Pipeline 引擎

`pipeline.py` 負責管理步驟堆疊（stack），每次「套用」會將（方法 id、參數快照）推入堆疊，並對當前影像套用對應函式。「復原」則從堆疊取出並從頭重播。

### 影像資料流

```
上傳影像
   └→ current_image（每次套用後更新）
          └→ fn(current_image, params) → preview_image（即時顯示）
```
